Algorithm
---------
Status of members (e.g.: event):
A (Accepted), R (rejected), I (invited), K (kicked, maybe suspended/silenced), L (left), V (Verified),
T (Timed-Out - invited, but not respond within a built-in timeframe), LL (late leave - payment is due),
W (on waiting list), LD (left deleted), D (deleted), P (pending), Selected (S)

if someone kicked out or late left then global score drops, not the mutual score, and it defines visibility
time decay, if someone didn't meet with each other in 2 years time, can drop visibility (?? not sure, how to manage)
if someone is inactive on the app, will be set "inactive" / host only mode automatically and the app asks to set it visible manually for the user
the graph stays intact - inactive does mean logout from every event, and the frieds also do not see it - maybe Friends Only also needed

if some requirement is not fulfilled on event, the event will be stopped (admin will be offered to stop or postpone etc.)
notify a day before or so (maybe can be configurable not sure)

if too many people rejects the invitation for a room, then the algorithm re-run for the left overs
If somebody lefts, nobody replaces in random chat (avoid algorithm complexity - or simply use left overs)
